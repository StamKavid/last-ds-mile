**Sanity check against domain expectations:** the top two drivers ‚Äî how long someone's
been a customer, and what contract they're locked into ‚Äî are exactly the two strongest
individual patterns already found in `02-explore.md`'s EDA (the tenure-decay curve and
the 15x churn-rate spread across contract types), and stable across both re-seals of this
benchmark even though the exact importance values and lower ranks shifted with the
stratified-split fix (`TotalCharges` now appears in the top 4, where it wasn't previously
prominent). This consistency of the *top* two ‚Äî despite a genuinely different held
sample ‚Äî is itself a good sign: the model learned the same story a human analyst would
tell, not something opaque or split-sensitive at the top of the ranking.

**No leakage re-check needed:** nothing here approaches the "single feature has
near-perfect importance" Red Flag ‚Äî the top feature (`tenure`, 0.042) is a modest
fraction of total importance, consistent with a model using many weak-to-moderate
signals together rather than one dominant (and therefore suspicious) one.

**Cross-checked against the best single base learner** (`figures/08-feature-importance.png`,
generated by `benchmarks/telco-churn/generate_figures.py`): CatBoost alone (the strongest
individual base model) agrees with the ensemble on the top 2 (`tenure` > `Contract`), but
diverges below that ‚Äî the ensemble ranks `InternetService` and `TotalCharges` at #3/#4,
while CatBoost ranks `OnlineSecurity` and `MonthlyCharges` higher instead, with
`InternetService`/`TotalCharges` both notably lower standalone than in the blend. Unlike
this benchmark's previous re-seal (where CatBoost assigned uniformly higher importance to
nearly every feature) or house-prices' pattern (where some features vanished entirely
from one base model), this is a genuine *rank* disagreement below the top 2, not just a
magnitude difference ‚Äî a reminder that a single base learner's importance ranking, even
the best-scoring one, isn't a stand-in for the ensemble's, and which specific features
show up at rank 3+ is sensitive to both the exact held sample and which model is asked.

**SHAP summary** (`figures/08-shap-summary.png`): a beeswarm over 60 held rows, explaining
`P(Churn)` from the actual shipped ensemble's `predict_proba` as a black-box callable.
Confirms `Contract` and `tenure` as the two dominant drivers by both magnitude and
direction ‚Äî low `tenure` (blue) pushes the prediction toward higher churn probability,
consistent with `02-explore.md`'s tenure-decay finding. (Their relative order swapped
from the previous re-seal's SHAP plot ‚Äî `tenure` led there, `Contract` leads here ‚Äî the
same top-2-stable-but-order-sensitive pattern already noted for permutation importance
above; treat "which of these two is #1" as noise, not signal.)

**Hypothesis DAG** (`figures/08-hypothesis-dag.png`): a diagram of the reasoning already

**Update ‚Äî a real causal analysis now exists for this dataset.** Difference-in-differences
(the library originally checked) needs a treatment/control assignment *and* pre/post
periods, which this cross-sectional snapshot doesn't have. DoWhy and CausalML, which work
on purely observational cross-sectional data, do fit: see
[`../CAUSAL_ANALYSIS.md`](../CAUSAL_ANALYSIS.md) for a real backdoor-adjusted ATE estimate
(does signing a long-term contract *cause* lower churn, or is that just confounding by
who chooses long contracts?), refutation tests, and a CATE breakdown showing the effect
is roughly 2‚Äì3√ó larger for new and fiber-optic customers than for others. Genuinely
complementary to the ranking model here, not a replacement for it ‚Äî this is kept as a
supplementary analysis, not a numbered pipeline stage, since it's a different question
("what intervention would help," not "who will churn").
