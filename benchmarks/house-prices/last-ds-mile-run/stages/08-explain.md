**Sanity check against domain expectations:** the top two drivers — living area and
overall quality rating — are exactly what a real-estate professional would name first,
and match `02-explore.md`'s bivariate correlations (`GrLivArea` r=0.70, `OverallQual`
r=0.82) almost feature-for-feature. No single feature dominates implausibly (top
importance is 0.055, not the near-1.0 "one feature explains everything" pattern
`ds-method`'s Red Flag warns about) — nothing here looks like a leaked or proxy feature.

**No re-check needed:** every top-9 feature is a structural/quality/location attribute
already justified as known-at-prediction-time in `03-prep.md` — none of them are the
sale-transaction fields (`SaleType`/`SaleCondition`/`MoSold`/`YrSold`) that were
correctly excluded there. The `Neighborhood` feature's presence at rank 7 is consistent
with `07-evaluate.md`'s slice finding (older/cheaper neighborhoods are harder to price) —
the model has in fact picked up on location as a real signal, it just isn't enough to
fully close the low-tier error gap on its own.

**Cross-checked against the best single base learner** (`figures/08-feature-importance.png`,
generated by `benchmarks/house-prices/generate_figures.py`): the winning ensemble is a
`WeightedEnsemble_L2` over CatBoost/LightGBM/LightGBMXT/RandomForestMSE; CatBoost alone
(the strongest individual base model, `score_val` closest to the ensemble's) tracks the
ensemble's `GrLivArea`/`OverallQual`/`TotalBsmtSF` ranking closely, but shows near-zero
importance for `Neighborhood` and `GarageCars` where the ensemble does not — meaning that
signal comes through the *other* base models (LightGBM/RandomForest), not CatBoost. A
real, useful finding this cross-check surfaces, not decoration: no single base learner's
importance ranking alone would have shown `Neighborhood` mattering.

**SHAP summary** (`figures/08-shap-summary.png`): a beeswarm over 60 held rows, explaining
the actual shipped ensemble's `predict()` as a black-box callable (see the script for why
— AutoGluon's per-model internal preprocessing is fragile to reach into directly for a
specific base learner; explaining the real `predict()` is both simpler and more honest
about what this model card ships). Direction and magnitude both confirm the permutation
ranking: high `GrLivArea`/`OverallQual` push the prediction up, low values pull it down,
with no reversed or counter-intuitive direction for any top feature.

**Hypothesis DAG** (`figures/08-hypothesis-dag.png`): a diagram of the causal *reasoning*
already logged in `02-explore.md`'s hypothesis log (size/quality attributes bundling into
one underlying signal, `Neighborhood` as a separate driver) — not a fitted causal model.
See `BENCHMARKS.md` for why this run scoped "causal" to a hypothesis diagram rather than
actual causal inference (no treatment/intervention structure exists in this cross-sectional
dataset for a method like difference-in-differences to estimate).
